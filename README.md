## Project Overview

This repository demonstrates a **professional QA assessment** for the **Random Date Generator tool** ([Link](https://codebeautify.org/generate-random-date)).

The goal of this QA project is to evaluate the **functionality, usability, and robustness** of the tool through **manual testing**, identify bugs, and provide actionable feature suggestions.

---

## Objectives

- Validate correct date generation based on user inputs  
- Test handling of edge cases, invalid inputs, and large datasets  
- Assess output formatting and UI/UX responsiveness  
- Identify potential bugs, usability issues, and improvement opportunities  

---

## Key Deliverables

1. **Test Report** – `test-report.md`  
   - Includes testing objective, scope, environment  
   - Features tested and detailed test scenarios with results  
   - Links to identified bugs and UX observations  

2. **Bug / Issue Report** – `bug-report.md`  
   - Detailed functional, boundary, and usability bugs  
   - Feature improvement suggestions  
   - Severity and priority assigned for each issue  

---

## Features Tested

- Single and multiple random date generation  
- Start/End date validation  
- Date format selection and consistency  
- Leap year and boundary case handling  
- Mobile responsiveness and UI layout  
- Copy-to-Clipboard functionality  
- Stress testing with large number of dates  
- Negative testing with invalid inputs  

---

## Test Methodology

- **Testing Type:** Manual Testing  
- **Environment:** Google Chrome on macOS, MacBook  
- **Approach:**  
  1. Identify key functional areas of the tool  
  2. Define test scenarios including normal, boundary, negative, and edge cases  
  3. Execute tests and record results  
  4. Log bugs and observations  
  5. Provide UX and feature improvement suggestions  

---

## Observations & Insights

- The tool performs its core function efficiently  
- Minor UX improvements can enhance clarity for edge cases  
- Mobile responsiveness could be improved for small screens  
- Copy functionality is implemented correctly  
- Input validation can be strengthened for invalid date ranges and formats
